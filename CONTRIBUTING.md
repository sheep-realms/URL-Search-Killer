# 贡献规则指南

## 收录规则

我们只收录那些可能会被用于分享的 URL，这意味着以下 URL 不会被收录：

- 仅限特定非公共权限访问的内容；
- 没有分享价值的内容；
- 怎么看都不像是可以分享的内容。

## 提供内容

添加规则需要提供以下内容：

- `title`：规则名称，命名格式为：`网站名称/页面名称`、`网站名称/页面名称/子页面名称`...
    - 一般没有必要写这么长的面包屑，除非网站业务真的非常复杂。
- `match`：匹配规则，忽略协议头。使用星号 “`*`” 作为通配符。
    - 还可以使用正则表达式，但不建议这么做。正则表达式须包含协议头。
- `has_search`：是否需要匹配 URL 参数。（这并不是一个最佳实现，不能处理参数顺序调换的情况，请务过度依赖）
    - `false`：默认值，匹配规则必须完整匹配域名和路径。
        - 例如，`example.com/s` 只能匹配其字面值，不能匹配 `example.com/search`。
    - `true`：匹配内容将包含 URL 参数，且不再要求完整匹配。
- `keep`：保留的参数，字符串数组。
