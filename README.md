# URL Search Killer

你是否苦恼过别人给你分享的链接[^1]，包含了一串又臭又长的 URL 参数[^2]？这款小工具[^3]就是来解决[^4]这个问题的！

只需要将 URL 丢进去[^5]，填写一下需要保留的参数（幸运的话可以自动填写）[^6]，然后就能访问一个干净的 URL！

## 使用方法

- 方法一：直接在网页中填写。
    - 输出 URL 后会出现一个跳转按钮，其本质上是一个链接而不是什么基于 JS 的假链接[^7]，也不会传递 Referer 标头[^8]，请放心使用它[^9]。
- 方法二：将 URL 拖放到网页中。
    - 如果规则命中，它会直接在新标签页打开。
- 方法三：通过 URL 参数 `url` 传入经过 URL 编码的 URL。（这是什么绕口令吗？）
    - 如果规则命中，它会直接重定向。

## 贡献规则

内置规则中没有你需要的规则？请见 [CONTRIBUTING.md](CONTRIBUTING.md)。

[^1]: 分享的链接是指可以被浏览器直接访问的 URL 地址，而不是某些平台因不正当的竞争行为被迫推出的专用于他们自己平台的访问口令。
[^2]：有些 URL 参数包含了一大堆追踪遥测参数，这对你来说没有任何作用，甚至还有信息泄露的风险。他们本可以自己维护一个短链系统，却非要在 URL 参数里拉屎。
[^3]：至少目前看来，这个小工具确实很小，不像某些社交软件塞进去了一个你根本不会用到的游戏引擎。这个小工具会不会过于简陋了？不，没有设计就是最好的设计。
[^4]：这指的是一种解决方案的尝试，不是对软件功能的保证。
[^5]：不是让你物理意义上的丢进去，你可以复制链接粘贴到对应的文本框中，也可以直接用鼠标把链接拖拽进去。如果你连这都不会，我本想给你推荐一门计算机补习课程，但我突然意识到 —— 我去你是怎么来到这里的？！
[^6]：如果命中已有规则，就能自动填写。没命中？自己动手，丰衣足食。
[^7]: 尤其是那些使用了现代前端框架的网站，不规范使用链接的情形十分常见。使用 JS 实现链接跳转不仅会破坏浏览器默认行为，还会严重阻碍无障碍辅助技术准确识别链接。
[^8]: 某些网站会根据 Referer 标头阻止你访问他们不愿意公开分享但又没有理由删除的内容。
[^9]: 这里是指这个按钮会按照预期的行为运行，完全没有 JS 脚本偷偷给链接塞追踪参数，也不会进入一个页面告诉你即将离开网站请注意安全。所以请你自行判断你要访问的链接是否安全，但你都用上这个工具了想必你应该知道你在干什么。
